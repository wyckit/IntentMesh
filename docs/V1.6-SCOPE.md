# IntentMesh v1.6 — "Production-Adjacent" Scope

> **Don't execute language. Execute verified intent.**

v1.0–v1.5 proved the *architecture* (the safety kernel, the three guards, the signed audit) and made
the integration seams *real* (MCP stdio + HTTP/SSE, OpenAPI JSON/YAML, SMTP + OAuth device flow) and
*hardened* (env-keyed audit, fail-closed parsing, SSRF/path safety, provable consent). What it has
**not** yet shown is one *complete, externally credible path* exercised end-to-end by a real LLM.

v1.6 builds exactly that — and nothing else.

## The one bet

Build a single complete path and make every link real and tested:

```
LLM proposer → typed intent → policy gate → real MCP/OpenAPI tool → approval
            → signed, persisted audit → replay → benchmark comparison
```

This is the difference between "a kernel that *could* sit in front of an agent" and "a kernel that
demonstrably *does*, with the receipts."

## In scope (v1.6)

| # | Deliverable | Why it's the credible next step |
|---|-------------|--------------------------------|
| 1 | **`LlmIntentProposer`** — a real LLM behind the `IIntentProposer` seam, constrained to the registered `Kinds` (Translation-Drift), fully **untrusted** (still gated). Real `AnthropicLlmClient` (zero-dep `HttpClient`, `ANTHROPIC_API_KEY`), with the LLM transport injectable so logic is testable offline. | Proves the headline: "language proposes; only typed, validated intent executes" — *with an actual language model proposing*, not a rule resolver. |
| 2 | **Malformed/ambiguous proposer tests** | A real LLM hallucinates unregistered kinds, emits malformed JSON, and can be steered by injected text. The kernel must stay fail-closed against all three. |
| 3 | **Run/audit persistence** — `IRunArtifactStore` + `FileRunArtifactStore` writing the five signed artifacts per run; **replay** verifies the signature and re-derives a byte-identical bundle. | An audit you can't persist and re-verify later isn't an audit. File-based first; the seam allows a DB/blob sink later. |
| 4 | **One full external integration slice** — the path above wired end-to-end (real proposer → gate → real MCP/OpenAPI tool → approval → persisted signed audit → replay), as a runnable demo **and** an in-process E2E test. | This is the artifact that makes the pitch undeniable. |
| 5 | **Live benchmark harness + report** — run naive agent / MCP-gated / IntentMesh head-to-head over the attack scenarios through the **real** pipeline; publish a short, honest comparison. | Moves IntentBench from "deterministic baseline models" toward a live, reproducible head-to-head. |
| 6 | **README claims discipline** — separate *proven* (has a passing test) from *future platform* claims. | Credibility rule: every proven claim is backed by a test that would fail if the claim were removed. |

## Explicit non-goals (v1.6)

- **No new demo domains.** Do not broaden to more agent verticals (travel, finance, ops, …). Depth over breadth.
- **No production key-management infra.** The `IAuditKeyProvider` env seam is enough for v1.6; KMS/HSM wiring is later.
- **No RSRM hot-load, no policy DSL compiler.** Those remain future (see [ROADMAP.md](ROADMAP.md) and [POLICY-AUTHORING.md](POLICY-AUTHORING.md)).
- **No multi-tenant / authn-z / deployment hardening.** v1.6 is "production-adjacent," not "in production."

## Invariants this scope must preserve

1. The proposer is **untrusted** — an LLM proposal never gains authority; it is gated like any other.
2. **Translation-Drift**: the proposer may only emit registered `Kinds`; unregistered kinds are dropped to `Unsupported`, never executed.
3. **Fail-closed** everywhere new input is parsed (LLM JSON, persisted artifacts, specs).
4. The verifier stays **prompt-blind and deterministic**.
5. **Zero-dependency** ethos holds — BCL only (`HttpClient`, `System.Text.Json`), no new NuGet.
6. Real I/O (LLM, network, disk) sits behind an **injectable seam** so the test suite runs offline and deterministically.

## Acceptance criteria

- An LLM proposal to email an attacker is still **gated** (Confirm/Block) — proven by a test.
- A hallucinated/unregistered kind and malformed LLM output are handled **fail-closed** — proven by tests.
- A full run can be **persisted, re-loaded, and replayed** with a verifying signature — proven by a test.
- The end-to-end slice runs in-process (scripted LLM + in-process MCP server) green in CI; the real-API and real-server paths are env-gated.
- A published comparison report shows naive / MCP-gated / IntentMesh outcomes, with each IntentMesh claim backed by a passing test.
- Full suite green, 0 warnings; README distinguishes proven from future.
